CSS Box Model

Although I'm a little familiar with HTML and CSS, I learned quite a bit from the presentation that I missed this past Saturday (Dec 1st).

The first new concept for me was the CSS box model, which describes the container-like composition of HTML elements. The image below demonstrates this model (note the boxes comprising the content, padding, border, and margin).

It is useful to use more specific tags, when appropriate, as they hold more meaningful information. An example of this would be using the header tag instead of simply using a div tag. Additionally, it is best to approach CSS section by section rather than tackling the entire site at once.

When it comes to defininig the parameters of elements like padding and margin, you always move clockwise (starting from the top): top, right, bottom, left. If you provide a single number, that number will apply to all four sides of the element. If you provide two numbers, the first will apply to the top and bottom, while the second will apply to the left and right sides.

The order of your code is important, as children will inherit information from the parent. Additionally, the latest line of code take precedence over previous lines. Thus, if your code looks like this:


div {

margin: 0;

margin-bottom: 20px;

}


Then the margin bottom will be 20px instead of 0px.


CSS Syntax

In CSS, classes are called upon using .classname, IDs are called upon using #IDname, and elements simply use their name. It is best practice to use dashes to distinguish words in class selectors (e.g. .my-div). This is also known as kabob case. To distinguish words in ID selectors, use camelcase (e.g. #myDiv).

Floating

Some element srae block elements by default, while others are inline (This can always be overridden using CSS). Floating transforms block elements to inline elements. Floats are necessary for building web layouts.

Sometimes you will need to clear a float so that it does not interfere with your layout. This is done using clearfix as a class.

The pseudoelement ::after is used to style specific parts of an element. It adds a hidden HTML element after the content of the .clearfix element and clears the float. Using an * indicates that the parameters should be applied to everything.


CSS Positioning

The positioning in CSS is static by default. This means that the element will stick to the normal page flow. Switching the boxes to relative will nudge the boxes in relation to their "original" location (top, left, bottom, or right). If you make the positioning absolute, then the boxes will be positioned relative to the nearest positioned ancestor. A fixed position will have exact coordinates in the browser window.

Layering with the Z-index allows you to layer elements on top of each other when they are positioned.
